## 2. 全アプリ共通の基本設定

Gemini Canvasで作成する際、**すべてのアプリで共通して以下の条件**を最初に入力しています。

これにより、学校現場で最も重要な「**セキュリティ**」と「**使いやすさ**」を担保しています。

### 基本プロンプト（すべてのアプリ共通）

```
以下の要件を満たすWebアプリを作成してください。

【基本仕様】
- HTML5、CSS3、JavaScript（Tailwind CSS等のCDNも利用可）
- 1ファイルで完結するHTMLファイル
- サーバーへの通信は一切行わない
- すべての処理をブラウザ内で完結させる
- 教員が使いやすい、シンプルで見やすいUI

【理由】
- 1ファイル構成 → インストール不要、USBや共有フォルダで配布可能
- ローカル完結 → 児童生徒の個人情報が外部に送信されない
- Tailwind等のCDN利用 → 1ファイルで美しいUIを実現
- サーバー不要 → 学校のネットワークに依存しない、どこでも動作
```

### なぜこれが重要か？

| 要件 | 学校現場でのメリット |
|------|---------------------|
| 1ファイル | 「インストール不要」で即使える、ファイル共有が簡単 |
| ローカル完結 | 個人情報保護、セキュリティポリシーに抵触しない |
| Tailwind CSS等 | デザインスキルがなくても美しいUIが作れる |
| サーバー不要 | 校内LANでも、オフラインでも、どこでも動作 |

## 3. カテゴリ1: 画像・映像・AI処理（4アプリ）

### ① 全自動・顔モザイクメーカー

**目的:** ウェブカメラやアップロードされた動画から自動で顔を検出し、モザイク処理を行う。

**主な機能:**
- face-api.jsを使った顔検出（AIライブラリ）
- リアルタイムカメラ映像への適用
- モザイク/ぼかし処理を選択可能
- 処理後の動画をダウンロード
- サーバーにアップロードせず完全にローカル処理

#### Gemini Canvasへのプロンプト

```
【全自動・顔モザイクメーカー】

以下の要件を満たすWebアプリを作成してください。

【基本仕様】
- HTML5、CSS3、JavaScriptのみで構築
- face-api.jsライブラリ（CDN）を使用して顔検出
- ウェブカメラまたは動画ファイルアップロードに対応
- 検出した顔にモザイク処理またはぼかし処理を適用
- 処理済み動画をMP4形式でダウンロード可能
- サーバーへの通信は一切行わず、すべてクライアント側で処理

【UI要件】
- 「カメラを使う」「動画をアップロード」のボタン
- 処理の進行状況を表示するプログレスバー
- モザイク強度のスライダー調整機能
- 「処理開始」「ダウンロード」ボタン
- 顔検出の精度を「低・中・高」で選択可能

【デザイン】
- Tailwind CSSを利用
- 明るく親しみやすいカラー
- スマホ・タブレット対応のレスポンシブデザイン
```

#### 作成時の注意点

⚠ **Chrome推奨:** face-api.jsは一部のブラウザで動作制限あり  
⚠ **重い処理:** 動画が長いと時間がかかるため、処理中メッセージを表示  
⚠ **顔検出精度:** 横顔や遠距離は検出できない場合がある  
⚠ **ファイルサイズ:** モデルデータをCDNから読み込むため初回は読み込み時間が発生  

#### カスタマイズのポイント

- **モザイクサイズを変更したい** → スライダーの値をfillRect()のサイズに連動
- **顔の枠を表示したい** → strokeRect()で検出座標に四角形を描画
- **音声はそのまま残したい** → MediaRecorderのaudio trackをそのまま保持

#### 教員・ICT支援員向けアドバイス

✅ **使い方の例:**
- 学芸会・運動会の動画を保護者向けに公開する前に自動でモザイク
- 学級通信に掲載する写真の顔ぼかし処理
- オンライン授業の録画で映り込んだ他の児童の顔を隠す

✅ **注意点:**
- 完璧な検出ではないので、必ず最終確認を行う
- 検出漏れがある場合は「他のモザイクアプリ」と併用
- 処理前のオリジナルファイルは必ずバックアップ

### ② FormMirror（動作分析・比較アプリ）

**目的:** 運動フォーム、ダンス、実験動作などを動画で撮影し、お手本と比較しながら分析する。

**主な機能:**
- 複数動画の同時再生・一時停止
- スロー再生（0.25x, 0.5x, 0.75x）
- 1フレームずつのコマ送り
- 画面分割表示（左右比較、上下比較）
- スマホで撮影した動画を直接アップロード可能

#### Gemini Canvasへのプロンプト

```
【FormMirror - 動作分析・比較アプリ】

以下の要件を満たすWebアプリを作成してください。

【基本仕様】
- HTML5、CSS3、JavaScriptのみで構築
- 複数の動画ファイル（MP4、MOV等）をアップロード可能
- 動画を同期再生・一時停止
- スロー再生機能（0.25倍、0.5倍、0.75倍、等倍）
- 1フレームずつコマ送り・戻しボタン
- 画面分割表示（横並び、縦並び、重ね合わせ）
- 再生位置の同期機能
- サーバーにアップロードせず、完全にローカル処理

【UI要件】
- ドラッグ＆ドロップでファイルアップロード
- 「お手本動画」「生徒動画」のラベル表示
- 再生・一時停止・コマ送りの直感的なボタン
- 分割表示切り替えボタン
- スロー再生速度スライダー

【デザイン】
- Tailwind CSSを使用
- 体育教員や部活動顧問が使いやすいシンプルなUI
- スマホ・タブレット対応
```

#### 作成時の注意点

⚠ **動画の同期:** 異なるフレームレートの動画は完全同期できない場合あり  
⚠ **重い処理:** 4K動画などは重くなるため、HD以下を推奨  
⚠ **スマホ撮影:** 縦向き/横向きの自動回転に注意  
⚠ **フレーム精度:** コマ送りはブラウザ依存のため厳密な1フレーム単位ではない  

#### カスタマイズのポイント

- **角度を測りたい** → Canvas APIで線を描き込む機能を追加
- **スローモーション録画** → MediaRecorderでスロー再生中を録画
- **比較動画の透明度調整** → CSSのopacityやmix-blend-modeを利用

#### 教員・ICT支援員向けアドバイス

✅ **使い方の例:**
- 体育: 跳び箱、マット運動、走り方のフォームチェック
- 音楽: リコーダー演奏の指使い確認
- 理科: 実験手順の動作確認
- 部活: バスケのシュートフォーム、野球のバッティングフォーム分析

✅ **注意点:**
- プライバシー配慮: 他の児童が映り込んでいないか確認
- 保護者同意: 動画を使用する際は事前許可を取る
- 長時間動画は分割: 必要な部分だけトリミングしてから使用

### ③ 万能PNGコンバーター

**目的:** PDF、Word、Excel、PowerPointなどのファイルを画像（PNG）に一括変換。

**主な機能:**
- PDFをページごとに分割してPNG化
- Officeファイル（docx, xlsx, pptx）をPNG化
- 一括変換・個別ダウンロード
- プレビュー機能
- DPI設定可能（解像度調整）

#### Gemini Canvasへのプロンプト

```
【万能PNGコンバーター】

以下の要件を満たすWebアプリを作成してください。

【基本仕様】
- HTML5、CSS3、JavaScriptのみで構築
- pdf.js（CDN）を使用してPDF描画
- mammoth.js（CDN）でdocx、xlsx-to-jsonでxlsx対応
- 各ページ/シートを個別にPNGとしてダウンロード
- 複数ファイル一括処理
- サーバーにアップロードせず、完全にローカル処理

【UI要件】
- ドラッグ＆ドロップで複数ファイルアップロード
- プレビュー表示機能
- DPI設定スライダー（72, 150, 300dpi）
- 「すべてダウンロード」「個別ダウンロード」ボタン
- 進行状況表示

【デザイン】
- Tailwind CSSを利用
- ファイル一覧をカード形式で表示
- レスポンシブ対応
```

#### 作成時の注意点

⚠ **ライブラリの互換性:** pdf.js は PDF 1.7 まで対応（最新の PDF 2.0 は非対応の場合あり）  
⚠ **複雑なレイアウト:** Excel の複雑な数式・グラフは正しく変換されない場合あり  
⚠ **メモリ使用量:** 大量ページの PDF は重くなる → 一度に処理するページ数を制限  
⚠ **ブラウザ制限:** Safari で一部ライブラリが動作しない可能性 → Chrome 推奨  

#### カスタマイズのポイント

- **背景を透明にしたい** → Canvas の clearRect() で背景を透過
- **ページ範囲を指定したい** → 開始・終了ページの入力フォーム追加
- **ZIP 圧縮してダウンロード** → JSZip を使って一括ダウンロード

#### 教員・ICT支援員向けアドバイス

✅ **使い方の例:**
- PDF教材をタブレットの壁紙にして視覚支援
- Word文書をPNG化して授業スライドに貼り付け
- Excelの表をそのまま画像化して配布プリントに

✅ **注意点:**
- 著作権確認: 変換するファイルの利用許諾を確認
- 解像度設定: 印刷用は300dpi、画面表示用は150dpiが目安
- ファイル名: 自動生成されるファイル名を分かりやすく変更

### ④ QR Generator（一括QRコード作成）

**目的:** URLやテキストを一括でQRコードに変換し、印刷・配布できる。

**主な機能:**
- CSV一括読み込み
- テキスト入力でQRコード生成
- サイズ調整（100px〜500px）
- 誤り訂正レベル設定（L/M/Q/H）
- A4印刷レイアウト最適化
- 個別ダウンロード・一括PNG保存

#### Gemini Canvasへのプロンプト

```
【QR Generator - 一括QRコード作成ツール】

以下の要件を満たすWebアプリを作成してください。

【基本仕様】
- HTML5、CSS3、JavaScriptのみで構築
- qrcode.js（CDN）を使用してQRコード生成
- テキスト入力、CSV一括読み込み対応
- サイズ調整（100〜500px）
- 誤り訂正レベル選択（L、M、Q、H）
- 生成したQRコードを個別・一括でPNGダウンロード
- A4用紙に最適化した印刷レイアウト
- サーバーにアップロードせず、完全にローカル処理

【UI要件】
- テキストエリア（複数行入力）
- CSVファイルアップロード
- サイズスライダー
- 誤り訂正レベル選択ボタン
- プレビュー表示エリア
- 「すべてダウンロード」「印刷」ボタン

【デザイン】
- Tailwind CSSを使用
- QRコードカード形式で一覧表示
- レスポンシブ対応
```

#### 作成時の注意点

⚠ **QRコードサイズ:** スマホで読み取る場合は200px以上推奨  
⚠ **URLの長さ:** 長すぎるURLはQRが複雑になり読み取りにくい → 短縮URLサービス利用推奨  
⚠ **誤り訂正レベル:** Hは最も高い誤り訂正だが、QRが大きく複雑になる  
⚠ **CSV形式:** 1行1URLの形式を想定（複雑なCSVは対応しない）  

#### カスタマイズのポイント

- **ロゴを埋め込みたい** → 中央にロゴ画像を配置（誤り訂正レベルをH推奨）
- **色を変更したい** → qrcode.jsのcolorDark/colorLightオプションで設定
- **連番QRを生成** → JavaScriptで連番URLを自動生成

#### 教員・ICT支援員向けアドバイス

✅ **使い方の例:**
- デジタル教材へのアクセスQRを配布プリントに印刷
- 保護者向けお便りにGoogleフォームのQRを掲載
- 校内掲示物に施設案内のQRコードを貼る
- 図書室の本にQRをつけて感想フォームへ誘導

✅ **注意点:**
- 短縮URL推奨: 長いURLは短縮サービス（Bitly, TinyURL等）で短くする
- 印刷確認: 実際に印刷してスマホで読み取りテストを行う
- 配布範囲: 個人情報が含まれるURLは配布範囲に注意

## 4. カテゴリ2: ファイル処理・変換（4アプリ）

**目的:** 教室の座席配置、掲示物レイアウトなどを視覚的にシミュレーションする。

**主な機能:**

#### Gemini Canvasへのプロンプト

```
【教室シミュレーター】

以下の要件を満たすWebアプリを作成してください。

【基本仕様】
- HTML5、CSS3、JavaScriptのみで構築
- Canvas API または SVG を使用して描画
- 机、椅子、教卓、ロッカー、掲示板などのアイコンをドラッグ配置
- グリッド表示でスナップ機能
- レイアウトをJSON形式で保存・読み込み
- PNG画像としてエクスポート
- サーバーにアップロードせず、完全にローカル処理

【UI要件】
- 左サイドバーにアイコンパレット
- 中央に教室キャンバス
- 右サイドバーに設定パネル（座席数、サイズ等）
- 「保存」「読み込み」「画像保存」ボタン
- アンドゥ・リドゥ機能

【デザイン】
- Tailwind CSSを使用
- アイコンはFont Awesome利用
- レスポンシブ対応（タブレット推奨）
```

#### 作成時の注意点

⚠ **座標管理:** ドラッグ時の座標計算が複雑 → グリッド単位で管理すると楽  
⚠ **保存形式:** JSON形式で保存 → localStorageまたはファイルダウンロード  
⚠ **画像品質:** Canvas の toDataURL() で PNG 出力 → 高解像度にするとファイルサイズ増  
⚠ **タッチ操作:** スマホではドラッグしにくい → タブレット以上推奨  

#### カスタマイズのポイント

- **実寸サイズ計算:** 1マス=1mなど単位を設定
- **座席番号の自動割り当て:** 座席順に番号を表示
- **写真の貼り付け:** 実際の教室写真を背景に配置

#### 教員・ICT支援員向けアドバイス

✅ **使い方の例:**
- 新学期の座席配置シミュレーション
- 行事の会場レイアウト検討
- 特別教室の備品配置計画
- 避難経路の図示

✅ **注意点:**
- 実測推奨: 実際の教室サイズを測ってから入力
- 保存必須: ブラウザを閉じると消えるため、必ず保存
- 印刷テスト: A4印刷して実際のスケール感を確認

  1. ビジュアルタイマー
  2. 座席表・班編成
  3. 名簿管理
  4. 出欠記録
  5. 係活動チェックリスト
- すべてのデータをlocalStorageに保存
- CSV入出力対応

#### 作成時の注意点

⚠ **データ消失リスク:** localStorage はブラウザのキャッシュクリアで消える → 定期的にCSV出力推奨  
⚠ **容量制限:** localStorage は約5〜10MB → 大量データは外部保存  
⚠ **ブラウザ依存:** 別のブラウザではデータが引き継がれない  
⚠ **同時編集不可:** 複数端末で同じデータを共有できない  

#### カスタマイズのポイント

- **クラウド同期:** Google Drive APIで自動バックアップ
- **通知機能:** 係活動の期限をブラウザ通知
- **統計表示:** 出席率グラフを Chart.js で可視化

#### 教員・ICT支援員向けアドバイス

✅ **使い方の例:**
- 朝の会でタイマー＋出席確認を同時進行
- 班活動の座席表を瞬時に表示
- 係活動の進捗を一目で確認
- 学期末に出席データをCSV保存して成績処理

✅ **注意点:**
- バックアップ必須: 学期末など重要な時期は必ずCSV保存
- 個人情報管理: 名簿データの取り扱いに注意
- 使用端末の固定: 同じブラウザ・端末で使用を推奨

## 9. プロンプトのコツ

### ✅ DO（こうする）

- **数値を明確に:** 「大きく」ではなく「24px以上」
- **条件を具体的に:** 「残り時間が1分以下なら黄色、0秒で赤色」
- **例を示す:** 「CSV形式は『番号,名前,性別』のヘッダー行必須」
- **ライブラリを指定:** 「Chart.jsを使用してグラフ表示」

### ❌ DON'T（こうしない）

- **曖昧な表現:** 「いい感じに」「適当に」「よしなに」
- **複数の要求を一度に:** 一度に10機能を要求すると混乱
- **矛盾した指示:** 「シンプルに」「機能豊富に」は両立困難
