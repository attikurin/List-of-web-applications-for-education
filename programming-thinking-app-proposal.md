# プログラミング的思考 育成アプリ 提案書

## 📋 概要

**アプリ名**: プログラミング思考トレーニングカード  
**英語名**: Programming Thinking Training Cards  
**想定URL**: https://attikurin.github.io/Programming-Thinking-Cards/

---

## 🎯 開発背景

### 文部科学省の方針
> 「発達の段階に即して、プログラミング的思考を育成すること。」

### 現場の課題
1. 既存アプリは小3以上対象が多く、低学年向けが不足
2. プログラミング言語に焦点が当たりすぎ、思考育成が後回し
3. 日常生活との結びつきが弱い
4. アンプラグド（紙ベース）教材が少ない
5. 教員の準備負担が大きい

---

## ✨ このアプリの特徴

### 1. 発達段階別の4つのモード
- **低学年（1-2年生）**: 絵カード中心、順序並べ替え
- **中学年（3-4年生）**: パターン発見、試行錯誤
- **高学年（5-6年生）**: フローチャート作成、条件分岐
- **中学生**: アルゴリズム設計、効率化

### 2. 5つの思考プロセスを育成
| プロセス | 内容 | 具体例 |
|---------|------|--------|
| 🧩 分解 | 問題を小さく分ける | 遠足の準備を「持ち物」「時間」「天気」に分解 |
| 🔍 パターン認識 | 共通点を見つける | 曜日ごとの時間割の共通パターン |
| 💡 抽象化 | 本質を抜き出す | 様々な動物の共通特徴 |
| 📋 アルゴリズム | 手順を整理する | カレー作りの手順 |
| 🐛 デバッグ | 間違いを修正 | 手順書の誤りを発見 |

### 3. 日常生活がテーマ
- 給食当番の手順
- 掃除の分担
- 運動会の準備
- 係活動の計画
- クラス遠足の計画

### 4. アンプラグド対応
- 印刷して紙カードとして使用可能
- デジタルとアナログの両方で学習
- タブレットでも快適操作

### 5. 教員サポート機能
- 指導案自動生成
- 評価基準の提示
- 学習到達度の可視化
- 各教科との連携ヒント

---

## 🎮 学習コンテンツ例

### 低学年（1-2年生）

#### カード1: 朝の準備カード
- **目標**: 順序立てて考える（アルゴリズム）
- **内容**: 6枚の絵カードを正しい順番に並べる
  - 起きる → 顔を洗う → 着替える → 朝ごはん → 歯を磨く → 学校へ
- **発展**: 「なぜこの順番?」を考える

#### カード2: 動物の仲間分けカード
- **目標**: パターン認識、抽象化
- **内容**: 12種類の動物を分類
  - 「空を飛ぶ」「水の中」「陸」
  - 共通点を見つける

---

### 中学年（3-4年生）

#### カード1: 給食当番の効率化
- **目標**: アルゴリズム、試行錯誤
- **内容**: 4人の役割分担を最適化
  - 複数パターンを試して比較
  - 「どの順番が一番早い?」

#### カード2: 間違い探しデバッグ
- **目標**: デバッグ
- **内容**: 手順書の3つの間違いを修正
  - 例: 「野菜を切る前に火にかける」

---

### 高学年（5-6年生）

#### カード1: 遠足計画フローチャート
- **目標**: 条件分岐、アルゴリズム設計
- **内容**: 天気による分岐を設計
  - 「もし〜ならば」の思考

#### カード2: クラス係活動の最適化
- **目標**: 効率化、抽象化
- **内容**: 5つの係活動の共通作業を抽出
  - 無駄を省いて効率化

---

### 中学生

#### カード1: アルゴリズムコンテスト
- **目標**: 複雑なアルゴリズム設計
- **内容**: 「最短ルート」「最大効率」問題
  - 複数の解法を比較

#### カード2: 社会問題解決カード
- **目標**: 実社会への応用
- **内容**: 「ゴミ削減」「エネルギー節約」
  - 問題を分解 → 解決策を設計

## 🛠️ 技術仕様

### 画面構成
```
┌─────────────────────────┐
│  ステージ選択           │ ← 発達段階別
├─────────────────────────┤
│  カード選択画面         │ ← テーマ選択
├─────────────────────────┤
│  学習画面               │ ← ドラッグ&ドロップ
├─────────────────────────┤
│  解答・フィードバック   │ ← 正誤判定・ヒント
├─────────────────────────┤
│  先生用管理画面         │ ← 評価・印刷
└─────────────────────────┘
```

### 主な機能
1. **ドラッグ&ドロップ**: カードを直感的に操作
2. **段階的ヒント**: 困ったら3段階でヒント表示
3. **印刷機能**: A4サイズで紙カードとして印刷
4. **学習記録**: ローカルストレージに保存
5. **評価基準**: 文科省の観点に準拠

### 技術スタック
- **HTML5/CSS3**: レスポンシブデザイン
- **JavaScript**: バニラJS（ライブラリ最小限）
- **Tailwind CSS**: スタイリング
- **LocalStorage**: データ保存
- **jsPDF**: PDF生成

## 📚 教育的価値

### 育成される力
1. **問題解決能力**: 複雑な問題を分解
2. **論理的思考**: 順序立てて説明
3. **試行錯誤力**: 失敗を恐れない姿勢
4. **メタ認知**: 思考過程を振り返る
5. **協働性**: グループで話し合う

### 評価の観点（文科省準拠）
- **知識・技能**: 手順を理解している
- **思考・判断・表現**: 効率的な方法を考える
- **主体的に学習に取り組む態度**: 試行錯誤を繰り返す

## 💡 実装の優先順位

### 🥇 優先度1: 低学年向け「順序カード」
**理由**: 
- 最もニーズが高い
- 実装が比較的シンプル
- すぐに授業で使える

**最小機能（MVP）**:
- 朝の準備カード（6枚）
- 動物の仲間分けカード（12枚）
- ドラッグ&ドロップ操作
- 正誤判定
- 印刷機能

### 🥈 優先度2: 中学年向け「パターン発見カード」
**理由**:
- プログラミング的思考の核心
- 試行錯誤を促進

**追加機能**:
- 複数解答パターン
- 段階的ヒント
- 学習記録

### 🥉 優先度3: 高学年向け「フローチャートカード」
**理由**:
- 中学校への接続
- より抽象的な思考

**追加機能**:
- 条件分岐
- ループ処理
- 評価基準表示
